## Common Issues & Solutions

### Issue: TypeScript errors on import
**Solution**: 
- Ensure all imports use correct paths
- Check that files exist in correct directories
- Run `npm install` after creating files

### Issue: TailwindCSS not working
**Solution**:
- Ensure `tailwind.config.js` has correct content paths
- Add to `src/index.css`:
```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

### Issue: Editor not showing
**Solution**:
- Check browser console for errors
- Verify extensions array is correct
- Ensure content prop is valid JSON
- Use `ensureBlockIds()` on content before passing

### Issue: Blocks not persisting
**Solution**:
- Verify `updatePageContent` is called
- Check localStorage in DevTools
- Verify store is updating (use React DevTools)
- Check that `onUpdate` callback exists

### Issue: Styles not applied
**Solution**:
- Ensure CSS file is imported in `BlockEditor.tsx`
- Check TailwindCSS is configured
- Verify class names are correct
- Use DevTools Inspector to debug

## Folder Organization Rules

**Follow these conventions:**

1. **Components**: Export React components only
   - Use PascalCase filenames: `BlockEditor.tsx`
   - One component per file (mostly)
   - Collocate CSS with component

2. **Hooks**: Export custom hooks only
   - Use camelCase filenames: `usePageEditor.ts`
   - Prefix with `use`

3. **Stores**: Export Zustand stores only
   - Use camelCase filenames: `pageStore.ts`
   - Include interface definitions

4. **Types**: Export TypeScript types only
   - Use camelCase filenames: `page.ts`
   - Group related types

5. **Utils**: Export pure utility functions
   - Use camelCase filenames: `blockHelpers.ts`
   - No side effects
   - Well documented

6. **Extensions**: Export TipTap extensions only
   - Use PascalCase filenames: `BlockIdExtension.ts`
   - One extension per file
